단위와 치수

몇몇의 단위는 다른 상황에서 다른 의미를 가집니다. 이번 챕터에서는 픽셀 밀도(Density)와 같은 개념과 더불어 장치 독립적(Device-independent) 픽셀과 확장 가능한(Scaleable) 픽셀의 사용법을 논의합니다.

픽셀 밀도

1인치에 꼭 들어맞는 픽셀의 수를 “픽셀 밀도(Pixel Density)” 라고 말합니다. 고밀도의 화면은 저밀도의 것보다 인치 당 더 많은 픽셀을 가집니다. 결과적으로, 버튼과 같은 UI 요소는 저밀도의 화면에서 물리적으로 더 크게 보이며, 고밀도의 화면에서 더 작게 보입니다.

화면 해상도는 한 디스플레이 안에 있는 픽셀의 수를 나타냅니다.

화면 밀도 = 픽셀 단위 화면 너비 (또는 높이) / 인치 단위 화면 너비 (또는 높이)

밀도 독립적 픽셀(dp)

"밀도 독립(Density independence)" 은 다른 밀도를 가진 화면들 위에서 UI 요소가 균일하게 표현되는 것을 말합니다.

밀도 독립적 픽셀(Density-independent Pixels, “dips”라고 발음)은 어떠한 화면 위에서도 균일한 크기로 크기를 변경하는 유연한 단위입니다. 안드로이드 어플리케이션을 개발할 때, 다른 밀도를 가진 화면 위에서 균일하게 요소를 표현하기 위해 dp를 사용하십시오.

1 dp는 160 dpi 화면 위에서 물리적 픽셀 하나와 같습니다. dp 계산법:

dp = (픽셀 단위 너비 * 160) / 화면 밀도

CSS를 작성할 때, dp 또는 sp로 명시된 어느 곳에든지 px를 사용하십시오. dp는 오직 안드로이드를 위한 개발에 사용되기 위해서 필요합니다.

확장 가능한 픽셀(sp)

안드로이드를 위한 개발의 경우, 확장 가능한 픽셀(Scaleable Pixels, sp)은 폰트를 위해 dp와 동일한 기능을 제공합니다. sp 하나의 기본값은 dp를 위한 기본값과 동일합니다.

sp와 dp의 주된 차이는 sp가 사용자의 폰트 설정을 보존한다는 것입니다. 접근성을 위해 큰 글자 설정을 한 사용자는 텍스트 크기 설정에 맞는 폰트 크기를 볼 것입니다.

dp로 레이아웃을 디자인하기

화면을 위한 레이아웃을 디자인할 때, dp 단위로 요소의 치수를 계산하십시오:

dp = (픽셀 너비 * 160) / 밀도

예를 들어, 320 화면 밀도에서 32 x 32 px 아이콘은 16 x 16 dp 와 같습니다.
