앱 내 검색

앱이 많은 양의 정보를 지원할 때, 사용자는 검색을 통해 내용물의 위치를 빠르게 찾아낼 수 있어야 합니다.

기본 검색은 다음을 포함합니다:

  1. 검색 텍스트 필드 열기
  2. 질의를 입력하고 제출
  3. 검색 결과의 집합을 표시

그러나, 검색 경험은 다음과 같은 것들의 제공을 통해 향상될 수 있습니다:

  • 음성 검색
  • 질의를 완전히 입력하기 전에 최근 사용자 질의를 기반으로 한 검색 히스토리 제안
  • 당신의 어플리케이션 데이터에서 실제 결과에 맞추어 자동 완성된 검색 제안

앱 내 검색을 위한 두 가지 주된 패턴이 존재합니다: 지속적인 검색(Persistent Search) 그리고 확장 가능한 검색.

지속적인 검색

검색이 당신의 앱의 주된 초점일 때 지속적인 검색(Persistent Search)을 사용하십시오.

동작:

검색 텍스트 필드는 삽입된 검색 박스의 내부에 보여지며, 초점을 얻을 준비를 하고 있습니다. 사용자는 음성 검색을 시작하기 위해서 마이크 아이콘을 터치할 수 있습니다.

초점을 얻을 때, 검색 필드는 검색 히스토리 제안을 보여 주기 위해서 확장합니다. 필요하다면, 화면 키보드 또한 보여질 것입니다.

어떠한 제안을 고르면 검색을 제출합니다. 위쪽에 화살표를 터치하면 검색에서 초점을 잃게 되며, 제안과 화면 키보드를 화면에서 없앱니다.

사용자가 질의를 입력하는 동안에, 검색 제안은 자동 완성으로 바뀝니다. 사용자가 글자를 입력하는 동안에, 제안은 걸러지고 정렬됩니다. 제안을 선택하거나 리턴 키를 누르면 검색을 제출합니다.

검색 박스에서 X는 질의를 초기화합니다.

검색 결과를 표시할 때, 검색 박스는 여전히 보이지만, 기본적으로 초점을 얻지 않습니다. 화면 키보드를 없애서 더 많은 결과가 보여질 수 있습니다.

검색 결과는 삽입된 검색 박스의 모양에 맞추며 각기 다른 유형의 결과를 수용하기 위한 카드 형식입니다.

확장 가능한 검색

검색이 당신의 앱의 주된 초점이 아닐 때 확장 가능한 검색(Expandable Search)을 사용하십시오.

동작:

검색 텍스트 박스 대신 툴바에 돋보기 아이콘을 표시하십시오.

검색 아이콘을 터치하면 툴바의 변형이 일어나고, 다른 내용물을 없애고 검색 텍스트 필드를 표시합니다. 만약 음성 검색이 지원된다면, 마이크 아이콘 또한 보여집니다.

검색 텍스트 필드는 자동으로 초점을 얻으며, 필요하다면 화면 키보드가 보여질 것입니다. 검색 히스토리 제안은 툴바의 아래에 보여질 수 있습니다. 제안 중 어떤 것을 선택하면 그 질의를 제출합니다.

위쪽 화살표를 터치하면 검색을 닫고 툴바의 원래 모습으로 돌아갑니다.

사용자가 질의를 입력하는 동안, 검색 제안은 자동완성으로 바뀝니다. 사용자가 글자를 입력하는 동안, 제안은 걸러지고 정렬됩니다. 제안을 선택하거나 리턴 키를 누르면 검색을 제출합니다.

검색 필드에 X는 질의를 초기화합니다.

검색 결과를 표시할 때, 툴바의 검색 버전은 여전히 남아있지만, 기본적으로 초점을 얻지 않습니다. 화면 키보드는 사라져서 더 많은 결과가 보여질 수 있습니다.

검색 결과는 카드 형식이고 툴바 아래에 페이지의 주 본문이 보여집니다.
