버튼

버튼은 사용자가 이것을 터치할 때 액션이 발생할 것이라는 것을 분명히 알려줍니다. 버튼은 당신의 앱의 색 테마에 따라서 디자인된 텍스트나 이미지 또는 양쪽 다로 구성됩니다.

버튼 유형

버튼에는 3가지의 표준 유형이 있습니다:

  • 플로팅 액션 버튼(FAB): 들려 있으며 누를 때 잉크 반응을 보여주는 둥근 머티리얼 버튼.
  • 레이즈드(Raised) 버튼: 들려 있으며 누를 때 잉크 반응을 보여주는 일반적인 직사각형 머티리얼 버튼.
  • 플랫(Flat) 버튼: 들려 있지는 않지만 잉크 반응을 보여주는 잉크로 만든 버튼.

버튼 스타일 고르기

버튼 스타일 고르기는 버튼의 수위(Primacy), 화면에서 컨테이너의 수, 그리고 화면 레이아웃에 달려 있습니다.

기능: 그것이 플로팅 액션 버튼이 되기에 충분히 중요하고 흔합니까?

차원(Dimension): 들어갈 컨테이너 그리고 z 공간 레이어가 당신의 화면에 얼마나 있는지에 따라서 레이즈드 또는 플랫을 선택하십시오. 화면에 많은 수의 개체 레이어가 있지 않아야 합니다.

레이아웃: 컨테이너 당 버튼은 주로 한 종류를 사용합니다. 혼합된 버튼은 중요한 기능을 강조하는 것과 같이 당신이 그에 대한 좋은 이유를 가지고 있을 때에만 사용하십시오.

사용법

다이얼로그에서 버튼

다이얼로그 내용물과 버튼을 통합하기 위해서 다이얼로그에 플랫 버튼을 사용하십시오.

  • 다이얼로그 텍스트 패딩: 24dp
  • 다이얼로그 버튼 컨테이너 높이: 52dp
  • 다이얼로그 버튼: 컨테이너 가장자리에서부터 8dp 패딩, 컨테이너 우측 정렬 (LTR 문자를 위해서)

인라인 버튼

당신의 레이아웃에 따라 레이즈드 또는 플랫 버튼을 고르십시오. 사용자가 쉽게 그것들을 찾을 수 있도록 플랫 버튼 주위에 패딩을 추가하십시오.

지속적인 푸터 버튼

당신의 앱이 지속적이고 사용자에게 손쉽게 가능한 액션을 필요로 한다면, 플로팅 액션 버튼을 사용하는 것을 고려하십시오.

그렇지 않으면, 쉽게 접근이 가능한 지속적인 푸터(Footer) 버튼 또한 고려해 보십시오.

지속적인 버튼 영역 안에 레이즈드 버튼은 결코 사용하지 마십시오.

디바이더의 추가로, 지속적인 푸터 버튼은 또한 스크롤 가능한 다이얼로그에 사용될 수 있습니다.

플랫 & 레이즈드 버튼

레이즈드 버튼

레이즈드 버튼은 주로 평평한 레이아웃에 차원(Dimension)을 추가합니다. 그것은 복잡하거나 넓은 공간에서 기능을 강조합니다.

플랫 버튼

지나친 레이어 사용을 피하기 위해서 툴바와 다이얼로그에 플랫 버튼을 사용하십시오.

플랫과 레이즈드 버튼의 상태

플랫 버튼은 머티리얼 위에 출력됩니다. 이것은 들어 올려지지는 않지만, 누를 때 색이 채워집니다.

레이즈드 버튼은 다른 시트에 의존한 머티리얼의 조각처럼 행동합니다—이것은 들어 올려지며 누를 때 색이 채워집니다.

버튼 텍스트는 대문자를 가진 언어에서 모두 대문자여야 합니다. 대문자가 없는 언어를 위해서, 플랫 버튼 위에 색이 있는 텍스트는 플랫 버튼을 일반 텍스트와 구별하게 만듭니다.

접근성

장애를 가진 사람들을 위한 사용성을 보장하기 위해서, 버튼의 높이가 36dp인지, 그리고 터치 가능한 대상이 48dp의 최소 높이를 가졌는 지를 확인하십시오.

버튼 엘리베이션

레이즈드 버튼은 2dp의 기본 엘리베이션(Elevation)을 가집니다. 데스크탑에서, 이 엘리베이션은 맴돌기(Hover) 상호작용을 위해 예약될 것입니다.

밀도

마우스와 키보드가 주 입력 방법일 때, 버튼 치수는 밀집한(Dense) UI를 수용하기 위해서 약간 줄여야 할 것입니다.

모션 안에서 버튼

토글 버튼

토글 버튼은 연관된 옵션을 그룹화 하기 위해서 사용될 것입니다. 어느 토글 버튼이 한 그룹의 일부분인 것을 전달하기 위해서 레이아웃과 공간을 정리하십시오.

초점을 얻은 상태와 눌려진 상태는 토글이 그룹의 일부분임을 나타내는 것을 보강할 것입니다. 예를 들어, 한 버튼 세그먼트가 초점을 얻었을 때, 초점은 동시에 그 세그먼트의 나머지에 표시될 것입니다.

토글 버튼 요구 사항:

  • 한 그룹 안에서 적어도 세 개의 토글 버튼이 존재할 것
  • 텍스트, 아이콘, 또는 둘 다가 있는 레이블 버튼일 것

다음의 조합이 권장됩니다:

  • 다중 선택 그리고 선택되지 않은 것들
  • 전용(Exclusive) 선택 그리고 선택되지 않은 것들
  • 전용 선택으로만 사용

아이콘 토글

아이템에 별을 추가하거나 제거하는 것과 같이, 아이콘은 선택되거나 선택 해제되기 위한 단일 선택을 허용하는 토글 버튼에 적절합니다.

아이콘은 앱 바, 툴바, 액선 버튼, 또는 토글 안에 위치하는 것이 가장 알맞습니다.

아이콘 토글은 그 터치 대상 경계를 넘는 경계가 있거나 또는 없는 잉크 표면 반응을 표시할 것입니다.

드롭다운 버튼

드롭다운(Dropdown) 버튼은 다양한 선택지 사이에서 선택합니다. 그 버튼은 현재 상태와 아래쪽 화살표를 보여줍니다. 예를 들어, 이용 가능한 상태는 문자열의 리스트, 팔레트, 또는 아이콘으로서 보여질 것입니다.

사용자가 버튼으로 상호작용 할 때, 메뉴는 버튼을 덮고 가능한 상태를 보여줍니다. 상태를 누르는 것은 메뉴를 사라지게 하고 새로운 상태를 보여주기 위해서 버튼을 갱신합니다.

드롭다운 내에서의 스크롤은 메뉴 스크롤과 동일합니다.

일반적인 오버플로우 드롭다운 버튼

이 버튼은 보여주는 상태가 없지만 화살표나 일반적인(Generic) 메뉴 버튼을 가진 드롭다운 버튼입니다. 버튼이 눌려졌을 때, 메뉴가 표시됩니다. 메뉴에서 옵션을 누르는 것은 일반적으로 그 옵션을 위한 추가적인 설정을 안내합니다.

세그먼티드 드롭다운 버튼

세그먼티드(Segmented) 드롭다운은 두 개의 부분을 가지고 있습니다: 현재 상태 그리고 드롭다운 화살표 아이콘. 현재 상태를 누르면 화면 내에서 발생하는 상태의 액션을 유발할 것입니다. 드롭다운 화살표를 누르면 모든 상태 옵션을 보여줄 것입니다. 하나를 선택하면 표시된 상태를 변경할 것입니다.

편집 가능한 세그먼티드 드롭다운 버튼

이 버튼은 폰트 크기 픽커와 같이 표시된 상태에 텍스트 편집이 가능한 곳이 있는 세그멘티드 드롭다운을 가집니다. 현재 상태를 누르면 상태의 액션을 실행하는 것과 표시된 상태를 편집이 가능하도록 만드는 것 모두를 수행합니다. 하나를 선택하는 것은 표시된 상태를 변경할 것입니다.
