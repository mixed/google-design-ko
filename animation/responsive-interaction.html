반응형 상호 작용

반응형 상호 작용은 사용자 입력으로 시기 적절하고, 사리에 맞으며, 마음에 드는 화면 반응을 생성함으로서 앱에 대한 깊은 탐구를 고취시킵니다. 각각의 상호 작용은 사려 깊고, 어쩌면 기발하겠지만, 결코 산만하지 않습니다. 내가 이 화면을 터치하면 무슨 일이 발생할까요? 그리고 나서 이 아이콘은?

차례

  • 사용자 입력
  • 표면 반응
  • 머티리얼 응답
  • 방사형 액션

사용자 입력

머티리얼 디자인에서, 앱은 다음과 같은 사용자 입력에 반응합니다:

  • 터치와 음성과 마우스와 키보드는 모두 동일하게 중요한 입력 방법입니다.
  • UI 요소는 유리막(장치의 화면) 뒤에 있다 할 지라도 만질 수 있는 것으로 보여집니다. 그 간격을 잇기 위하여 시각과 모션 자극은 즉각적으로 입력을 알리고, 직접적으로 조작하는 것 같이 보고 느끼는 방법으로 움직입니다.

반응형 상호 작용은 정보 전달 서비스에서부터 여러 시각과 촉각 반응을 사용하여 통신하는 경험까지 앱의 가치를 높입니다.

표면 반응

하나의 입력 이벤트에서, 시스템은 접촉하는 그 시점에 즉각적인 시각적 확인을 제공합니다: 터치를 위해 손가락 패드 아래에서, 음성을 위해 마이크에서, 또는 키보드 입력을 위해 적절한 필드에서.

이 접촉을 표현하기 위한 핵심 시각 메커니즘은 터치 리플(파문)입니다. 이 장치는 음성의 진폭이나 터치 압력과 같이 상당히 동적인 변수들로 터치 이벤트의 방법과 지속 기간을 표현합니다.

모범 사례

입력의 지점(손가락의 접촉 지점 또는 음성을 위한 마이크 아이콘)부터 시각적 반응 원을 만드십시오.

머티리얼 응답

잉크와 같은 표면 반응 뿐만 아니라, 머티리얼은 터치하거나 활성 상태를 나타낼 때 들어 올려질 수 있습니다. 터치에서 사용자는 새로 생성하거나, 존재하는 머티리얼을 변형하거나, 머티리얼을 드래그하거나 플링(밀어내기)하여 직접적으로 머티리얼 시트를 조작할 수 있습니다. 머티리얼은 선형적으로나 방사상으로(Radially) 크기 변경될 수 있습니다.

연관된 자료

뷰 상태 변경을 애니메이션하기
상태 변경 사이에 애니메이션을 보여주십시오.

발생지점

사용자가 새로운 머티리얼의 생성을 작동시킬 때, 그것은 입력 지점으로부터 시작하여 크기가 증가되어야만 합니다.

터치 시에 들어 올리기

카드 또는 구분할 수 있는 요소가 활성화되었을 때, 카드는 활성화 상태를 나타내기 위하여 들어 올려야 합니다.

방사형 액션

사용자 입력에 시작적인 반응들을 통해 사용자 입력에 명료성을 추가하십시오. 방사형 액션은 입력 지점으로 부터 퍼져나가는 잉크의 시각적 리플입니다.

입력 이벤트는 시각적입니다

입력 이벤트와 화면 액션 사이의 연결은 그들이 함께 묶여 시각적으로 표현되어야 합니다. 터치와 마우스를 위해서, 이것은 접촉 지점에서 발생합니다. 음성을 위해서는 마이크 아이콘입니다. 터치 리플은 터치가 어디서 또는 언제 발생하고 터치 입력을 받았다는 것을 인식했는 지를 나타냅니다.

입력을 액션에 연결하십시오

입력 이벤트에 의해 발생되는 전환 또는 액션은 시각적으로 입력 이벤트와 연결해야 합니다. 중심점 근처에서 리플 반응은 반응보다 빠르게 더 멀리 발생합니다.

예제:

  • 내용물의 변경의 순서: 이미지는 나타나거나 사라집니다.
  • 머티리얼 움직임의 순서: 카드는 화면 안으로 들어오거나 나갑니다.
